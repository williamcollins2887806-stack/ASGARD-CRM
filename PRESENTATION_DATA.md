## 1. РОЛИ И ДОСТУП

### 1.1. Количество ролей

В системе **11 активных ролей**. В коде определён объект `ROLES` (файл `auth.js:9-12`):

```js
const ROLES = {
  ADMIN:1, TO:1, PM:1, HR:1, BUH:1, OFFICE_MANAGER:1,
  DIRECTOR_COMM:1, DIRECTOR_GEN:1, DIRECTOR_DEV:1, PROC:1, WAREHOUSE:1
};
```

Ранее существовала роль `DIRECTOR` (единая), но она была удалена в Stage 25 (`app.js:121`):
```js
const DIRECTOR_ROLES = ["DIRECTOR_COMM","DIRECTOR_GEN","DIRECTOR_DEV"]; // legacy DIRECTOR removed in Stage 25
```

Также при регистрации создаётся промежуточный статус `PENDING` (не активная роль, а статус заявки).

## 2. ПОЧТА / EMAIL

### 2.1. Отдельная страница для почты

**Нет отдельной страницы** `/email` или `/mail` в сайдбаре. Email — это **интеграционный модуль**, встроенный в другие разделы.

### 2.2. Что есть в системе

**Backend** (`src/routes/email.js`):
- `POST /api/email/send` — отправка письма (to, subject, body, attachments)
- `GET /api/email/history` — история отправок пользователя
- `POST /api/email/test` — тестовая отправка (только ADMIN)
- `POST /api/email/settings` — сохранение SMTP-настроек (только ADMIN)

**Frontend** (`public/assets/js/email.js` — модуль `AsgardEmail`):

Модуль предоставляет модальное окно отправки email из CRM. Отправка вызывается из других модулей (счета, акты).

### 2.3. Шаблоны писем

5 встроенных шаблонов (`email.js:13-71`):

| Шаблон | Название | Описание |
|--------|----------|----------|
| `tkp` | Коммерческое предложение | Отправка ТКП клиенту |
| `invoice` | Счёт на оплату | С подстановкой номера, даты, суммы |
| `act` | Акт выполненных работ | С подстановкой номера и даты |
| `reminder` | Напоминание об оплате | С суммой и сроком оплаты |
| `custom` | Произвольное письмо | Пустой шаблон |

### 2.4. Возможности

- **Отправка писем из CRM** — да, через SMTP (Nodemailer)
- **Просмотр входящих** — нет (только SMTP, нет IMAP)
- **Вложения** — да, поддерживаются (base64)
- **Шаблоны с переменными** — да, подстановка `{invoice_number}`, `{total_amount}` и т.д.
- **История отправок** — да, таблица `email_log` в БД + `email_history` в IndexedDB
- **Настройки SMTP** — через БД (`settings.smtp_config`) или ENV переменные

### 2.5. Привязка к сущностям

- **К счетам** — `sendInvoice(invoiceId)` — автоматически подтягивает email контрагента по ИНН
- **К актам** — `sendAct(actId)` — автоматически подтягивает email контрагента по ИНН
- **К контрагентам** — email контрагента берётся из карточки (поле `customers.email`)
- **Напоминание об оплате** — `sendPaymentReminder(invoiceId)`

## 3. AI / МИМИР

### 3.1. Подробное описание возможностей

**Мимир — Хранитель Мудрости** (`public/assets/js/mimir.js`) — AI-ассистент на базе **YandexGPT**.

**Виджет-чат** — плавающая кнопка (🧙) в правом нижнем углу на всех страницах (кроме /welcome и /login). Разворачивается в чат-панель 450×600px.

### 3.2. Модули и страницы AI

Отдельной страницы AI-анализа **нет**. Мимир работает как виджет-чат + набор API-эндпоинтов:

**Backend** (`src/routes/mimir.js`):
- `POST /api/mimir/chat` — текстовый запрос к AI
- `POST /api/mimir/analyze` — анализ файлов (FormData)
- `POST /api/mimir/generate-tkp` — генерация коммерческого предложения
- `GET /api/mimir/tender-recommendation/:id` — рекомендация по тендеру
- `GET /api/mimir/finance-stats` — финансовая статистика
- `GET /api/mimir/works-analytics` — аналитика работ

### 3.3. Типы файлов для анализа

Принимаемые форматы (`mimir.js:321`):
```
.pdf, .xlsx, .xls, .csv, .png, .jpg, .jpeg, .txt
```
Лимит: **10 МБ** на файл.

### 3.4. Данные CRM, доступные Мимиру

Мимир автоматически подгружает контекст из БД с учётом роли:
- **Тендеры** — статистика, количество по статусам, за месяц
- **Работы** — общее количество, суммы контрактов
- **Финансы** — расходы, доходы, прибыль
- **Персонал** — данные сотрудников (для HR)
- Контекст текущей страницы передаётся автоматически (hash-роут)

### 3.5. Предустановленные запросы/подсказки

**Быстрые кнопки** (`mimir.js:310-314`):
- 📊 Статистика — `"Покажи статистику"`
- ⚠️ Просрочки — `"Какие счета просрочены?"`
- ⏰ Дедлайны — `"Ближайшие дедлайны работ"`
- 📝 Создать ТКП — открывает генератор ТКП

**Приветственные подсказки** (`mimir.js:565-567`):
- 📊 Тендеры — `"Сколько у нас тендеров?"`
- 🔍 Поиск — `"Найди работы по Газпром"`
- ❓ Помощь — `"Как добавить расход?"`

### 3.6. RBAC фильтрация в Мимире

Определено в `src/routes/mimir.js:22-28`:

| Роль | Доступ к данным |
|------|----------------|
| ADMIN, DIRECTOR_* | **Полный доступ** — все тендеры, работы, финансы, персонал |
| PM | Только **свои** тендеры и работы (фильтр по `pm_id`) |
| TO | Только тендеры (без финансов) |
| HR | Персонал (без финансов и тендеров) |
| BUH | Финансы (без тендеров) |
| Остальные | Ограниченная статистика |

### 6.1. Карточка контрагента

Файл: `public/assets/js/customers.js`

**Поля карточки:**
- ИНН (первичный ключ, 10 или 12 цифр)
- Название (краткое)
