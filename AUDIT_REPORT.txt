 НЕРЕШЁННЫЕ ПРОБЛЕМЫ (требуют решения владельца / живого тестирования)
═══════════════════════════════════════════════════════════════════════════════

  1. SECURITY HEADERS (LOW → нужно для production)
     Файл: src/index.js
     Проблема: Нет helmet middleware (X-Frame-Options, CSP, HSTS,
     X-Content-Type-Options)
     Решение: npm install @fastify/helmet + fastify.register(helmet, {...})
     Почему не исправлено: Требует npm install + конфигурация CSP под проект

  2. ТАЙМАУТЫ API (LOW → нужно для production)
     Файлы: src/services/ai-provider.js (Anthropic/OpenAI fetch),
             src/routes/mailbox.js:413, src/routes/email.js:83 (sendMail)
     Проблема: fetch() и sendMail() без timeout. При зависании внешнего
     сервиса запрос висит до дефолтного таймаута ОС/браузера.
     Решение: AbortController + setTimeout для fetch; connectionTimeout
     для nodemailer transport.
     Почему не исправлено: Нет npm-зависимостей, нужна конфигурация таймаутов

  3. WEBHOOK АВТОРИЗАЦИЯ (LOW)
     Файл: src/routes/integrations.js:810
     Проблема: POST /erp/webhook/:connection_id проверяет НАЛИЧИЕ токена
     в x-api-key / Authorization, но НЕ ВАЛИДИРУЕТ значение. Любая
     непустая строка пропускается.
     Решение: Добавить webhook_secret в erp_connections, при создании
     подключения генерировать UUID, при получении webhook — сравнивать.
     Почему не исправлено: Требует миграцию БД + изменение бизнес-логики

  4. XSS ЗАЩИТА (LOW → системная проблема)
     Файлы: 97 frontend JS файлов
     Проблема: Только 10 вызовов AsgardUI.esc() и 11 escapeHtml() на весь
     проект. Большинство модулей рендерят пользовательские данные через
     innerHTML без экранирования.
     Решение: Аудит всех innerHTML с пользовательскими данными, замена на
     AsgardUI.esc(). Приоритет: формы ввода, чаты, комментарии, имена.
     Почему не исправлено: Масштабная задача (100+ мест), высокий риск
     поломки при массовом изменении

  5. CORS В PRODUCTION (LOW)
     Файл: src/index.js:26
     Проблема: Если CORS_ORIGIN не задан в production — fallback на
     'https://asgard-crm.ru'. Но если кто-то установит CORS_ORIGIN='*'
     в .env — все ограничения снимутся.
     Решение: Убедиться что production .env содержит правильный CORS_ORIGIN.
     Добавить проверку: если production + *, то process.exit(1).
     Почему не исправлено: Конфигурация окружения, не код

  6. ТРАНЗАКЦИИ (LOW)
