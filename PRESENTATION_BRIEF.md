# АСГАРД CRM — Полное описание системы для презентации

> **Назначение документа:** Этот файл содержит исчерпывающее описание CRM-системы «АСГАРД» для подготовки презентации на 40–60 минут. Описаны все модули, роли, бизнес-процессы, интеграции, дизайн и технические детали.

---

## 1. ОБЩЕЕ ОПИСАНИЕ СИСТЕМЫ

**АСГАРД CRM** — корпоративная CRM-система компании «АСГАРД-СЕРВИС», предназначенная для полного цикла управления тендерами, проектами (работами), финансами, персоналом, оборудованием и внутренними процессами сервисной/промышленной компании.

**Ключевые характеристики:**
- **Тип приложения:** Progressive Web App (PWA) — работает в браузере, устанавливается как приложение на телефон/планшет/ПК
- **Офлайн-режим:** Полная поддержка работы без интернета через Service Worker и IndexedDB, с последующей синхронизацией
- **Язык интерфейса:** Полностью русский
- **Тематика оформления:** Скандинавская/Викинг-стилистика — все разделы названы в духе скандинавской мифологии (Зал Ярла, Сага Тендеров, Кузница Настроек и т.д.)
- **Количество модулей:** 87+ фронтенд-модулей, 21 серверный API-модуль
- **Покрытие тестами:** 47 E2E-тестов (Puppeteer)

**Стек технологий:**
- **Backend:** Node.js 20 + Fastify (быстрый REST API-фреймворк)
- **База данных:** PostgreSQL 15
- **Frontend:** Vanilla JavaScript SPA (без фреймворков — чистый JS для максимальной скорости)
- **Аутентификация:** JWT-токены + bcrypt-хеширование паролей + PIN-код (2FA)
- **Контейнеризация:** Docker + Docker Compose
- **Reverse Proxy:** Nginx с rate-limiting, gzip, кешированием
- **Интеграции:** Telegram Bot, Email (SMTP/IMAP), Манго Телеком (VoIP), AI-ассистент

## 2. ДИЗАЙН И ВИЗУАЛЬНОЕ ОФОРМЛЕНИЕ

### 2.1. Цветовая схема
Система использует **тёмную тему** «Corporate Red & Blue» (версия 26.0):

- **Основной фон:** Глубокий тёмно-синий (#0a1628) с радиальными градиентными свечениями красного и синего
- **Фон карточек:** Тёмно-синий (#0e1f3d) с рамками rgba(42, 59, 102)
- **Основной акцент (красный):** #dc2626 — кнопки, активные элементы, логотип
- **Вторичный акцент (синий):** #2563eb — ссылки, иконки, подсветка меню
- **Градиент:** Красно-синий gradient (135deg) — используется повсеместно для кнопок, рамок, заголовков
- **Текст:** Светло-серый (#f1f5f9) на тёмном фоне, мутный (#64748b) для второстепенного
- **Статусы:** Зелёный (#22c55e), Янтарный (#f59e0b), Красный (#dc2626), Синий (#2563eb)

### 2.2. Общий макет (Layout)
- **CSS Grid** из двух колонок: боковая панель (300px) + основной контент
- **Верхняя градиентная линия** (3px) вдоль всего верха экрана — анимированный красно-сине-золотой градиент, бесконечная анимация
- **Стеклянный эффект (glassmorphism):** Полупрозрачные панели с backdrop-blur
- **Скруглённые углы:** 8–20px радиусы на всех элементах
- **Тени с цветным свечением:** Красное и синее glow-свечение на элементах
- **Адаптивность:** На экранах < 900px сайдбар скрывается, появляется гамбургер-меню

### 2.3. Боковая панель (Sidebar)
- **Логотип** компании АСГАРД-СЕРВИС (56×56px) с красной рамкой
- **Название «CRM»** с градиентным текстом (красно-синий)
- **Руническая подпись** «ᚠᚢᚦᚨᚱᚲ» под логотипом
- **Навигационное меню** — вертикальный список пунктов, каждый с:
  - Иконкой (SVG, 20×20px) в квадратной подложке 38×38px
  - Названием (жирный, 13px) и описанием (мелкий, 11px)
  - Активный пункт подсвечен синим с вертикальной градиентной полосой слева
  - Hover-эффект: фон с градиентным свечением
- **Подвал сайдбара:** Имя текущего пользователя, его роль (в мини-бейдже), кнопка «Выйти»
- Кастомный тонкий скроллбар (6px)

### 2.4. Основная область контента
- **Верхняя панель (topbar):** Заголовок страницы (h1, 24px) + описание + бейджи/кнопки действий справа
- **Золотая линия-акцент** (120px) под заголовком
- **Цитаты-девизы** в стиле викингов на каждой странице (например: «Один видит всё. И твои дела — тоже.», «Рагнарёк подождёт — сначала сдай отчёт.»)
- **Загрузочный экран** с 30 рандомными викингскими цитатами

### 2.5. UI-компоненты
- **Таблицы (.asg):** Разделённые строки с закруглёнными углами (14px), тёмный полупрозрачный фон, отступы между строками 10px
- **KPI-карточки:** Сетка из 6 колонок, каждая с заголовком (11px), значением (22px, золотой цвет) и подзаголовком
- **Кнопки:** Градиентные красно-синие, с hover-эффектами, вариации: primary, ghost, red, success
- **Модальные окна:** Полноэкранный режим, закрытие по Escape/оверлею, плавные анимации
- **Тосты (уведомления):** Всплывающие сообщения с типами: success, error, warning, info
- **Графики:** SVG-based (без тяжёлых библиотек) — столбчатые, круговые, диаграммы Ганта
- **Шрифты:** system-ui, -apple-system, "Segoe UI", Roboto (системные, для быстрой загрузки)

### 2.6. Мобильная версия
- Гамбургер-меню с оверлеем
- Свайп от левого края для открытия меню
- Pull-to-refresh
- Минимальный размер кнопок 44×44px (стандарт Apple)
- Горизонтальная прокрутка таблиц
- Поддержка iOS Safe Area
- Вибрация при действиях (на поддерживаемых устройствах)

## 3. ВСЕ МОДУЛИ СИСТЕМЫ — ПОДРОБНОЕ ОПИСАНИЕ

### 3.1. МОДУЛЬ «ЗАЛ ЯРЛА» (Главная / Home)
**Путь:** `/home`
**Роли:** Все пользователи
**Описание:** Стартовая страница после входа. Показывает сводную информацию и навигационные порталы (быстрый доступ ко всем разделам). Каждый пользователь видит только те порталы, к которым у него есть доступ по роли. Отображает девиз: «Сталь и порядок. Пусть каждый день приносит добычу.»

### 3.2. МОДУЛЬ «ДАШБОРД РУКОВОДИТЕЛЯ» (Dashboard)
**Путь:** `/dashboard`
**Роли:** ADMIN, Директора
**Описание:** Аналитическая панель для руководства с KPI-метриками:
- **Тендеры:** Всего, выиграно, проиграно, в работе, конверсия (%)
- **Работы:** Всего, завершено, активных, проблемных
- **Выручка:** Контракты, план vs факт
- **Прибыль:** Расчёт, тренд
- **Расходы:** Рабочие, офисные, командировочные
- **Команда:** Активные сотрудники, заблокированные
- Помесячные графики (тендеры, работы, выручка по месяцам)
- Алерты по проблемным работам и заблокированным пользователям
- Экспорт данных в CSV

### 3.3. МОДУЛЬ «МОЙ ДАШБОРД» (Custom Dashboard)
**Путь:** `/my-dashboard`
**Роли:** Все офисные роли
**Описание:** Персональный настраиваемый дашборд. Каждый пользователь может выбирать виджеты, настраивать расположение и фильтры под себя.

### 3.4. МОДУЛЬ «ВОРОНКА ПРОДАЖ» (Funnel)
**Путь:** `/funnel`
**Роли:** ADMIN, TO, Директора
**Описание:** Канбан-доска для визуализации движения тендеров по стадиям. Показывает количество тендеров на каждом этапе: Новый → На просчёте → ТКП согласовано → Клиент согласился / Клиент отказался. Позволяет кликнуть на карточку для детального просмотра.

### 3.5. МОДУЛЬ «САГА ТЕНДЕРОВ» (Tenders)
**Путь:** `/tenders`
**Роли:** ADMIN, TO, Директора
**Описание:** Основной реестр тендеров компании. Полный CRUD (создание, чтение, обновление, удаление).

**Типы тендеров:** Тендер, Запрос предложений, Оценка рынка, Прямой запрос, Доп. объём

**Статусы тендеров:** Новый → На просчёте → ТКП согласовано → Клиент согласился / Клиент отказался / Другое / На распределении

**Функции:**
- Фильтры: по периоду, статусу, PM, типу, текстовый поиск
- Автоматическое обнаружение дубликатов (fuzzy matching — показывает 5 похожих)
- Передача тендера РП (PM) на просчёт с требованием документации
- Запрос на распределение (выбор PM)
- SLA-трекинг (дедлайны по документам, прямым запросам)
- Аудит-лог всех изменений
- Ссылки на тендерные площадки
- Планирование дат начала/окончания работ

### 3.6. МОДУЛЬ «КАРТА КОНТРАГЕНТОВ» (Customers)
**Путь:** `/customers`, `/customer/:inn`
**Роли:** ADMIN, TO, PM, Директора
**Описание:** Справочник клиентов/контрагентов.
- Поиск по ИНН (10 или 12 цифр) и названию
- Реквизиты: название, ИНН, КПП, ОГРН, адрес
- Контактные лица с должностями, телефонами, email
- Заметки и комментарии
- История контактов
- Связанные тендеры и работы по контрагенту

### 3.7. МОДУЛЬ «КАРТА ПОХОДА • ПРОСЧЁТЫ» (PM Calcs)
**Путь:** `/pm-calcs`
**Роли:** ADMIN, PM, Директора
**Описание:** Inbox руководителя проекта (РП). Показывает тендеры, переданные на просчёт данному PM. PM создаёт ТКП (технико-коммерческое предложение), используя встроенный калькулятор.

### 3.8. МОДУЛЬ «СОГЛАСОВАНИЕ» (Approvals)
**Путь:** `/approvals`
**Роли:** ADMIN, Директора
**Описание:** Центр принятия решений. Директор видит все ожидающие согласования:
- Премии рабочим
- Кадровые заявки
- Закупочные заявки
- Расходы на согласование

### 3.9. МОДУЛЬ «СОГЛАСОВАНИЕ ПРЕМИЙ» (Bonus Approval)
**Путь:** `/bonus-approval`
**Роли:** ADMIN, PM, Директора
**Описание:** Workflow согласования премиальных выплат рабочим.
**Статусы:** Черновик → На согласовании → Согласовано / Отклонено / Вопрос
**Процесс:**
1. PM завершает работу
2. PM создаёт запрос на премию с суммами для каждого рабочего и обоснованием
3. Директор рассматривает, может задать вопрос
4. При согласовании — бонусы попадают в учёт
5. При отклонении — PM получает комментарий и может пересоздать

### 3.10. МОДУЛЬ «КАРТА ПОХОДА • РАБОТЫ» (PM Works)
**Путь:** `/pm-works`
**Роли:** ADMIN, PM, Директора
**Описание:** **Крупнейший модуль системы (~72 КБ кода).** Полное управление проектами для PM.

**Статусы работ:** Новая → На просчёте → В работе → Подписание акта → Работы сдали | Проблема

**Функции:**
- Карточка работы: заказчик, тендер, даты, бригада, расходы
- Назначение сотрудников с ролью, датами, ставкой оплаты
- Трекинг: план vs факт (стоимость, сроки)
- Аванс и балансовые платежи от заказчика
- **Закрытие работы — визард с обязательной оценкой каждого сотрудника (1–10 баллов)**
- Оценка удовлетворённости клиента (1–10)
- Пакет документов: ТЗ, ТКП, письма, фото
- Запрос бонуса для бригады
- Комментарии и аудит-трейл
- Трекинг расходов по работе

### 3.11. МОДУЛЬ «СВОД КОНТРАКТОВ» (All Works)
**Путь:** `/all-works`
**Роли:** ADMIN, Директора
**Описание:** Общий реестр всех работ компании. Таблица с заказчиком, PM, статусом, таймлайном, суммами. Фильтры по периоду, поиску, PM, статусу. Вид Ганта для визуализации таймлайна. Показывает: аванс получен / баланс / остаток долга.

### 3.12. МОДУЛЬ «СВОД РАСЧЁТОВ» (All Estimates)
**Путь:** `/all-estimates`
**Роли:** ADMIN, Директора
**Описание:** Реестр всех ТКП (технико-коммерческих предложений) по компании. Фильтры по PM, тендеру, статусу.
**Статусы ТКП:** Черновик → Согласование ТКП → ТКП согласовано → Клиент согласился / Клиент отказался / На доработке

### 3.13. МОДУЛЬ «АНАЛИТИКА ЯРЛА» (Analytics / KPI)
**Путь:** `/analytics`
**Роли:** ADMIN, Директора
**Описание:** Объединённый KPI-дашборд по работам и деньгам. Включает два подмодуля:
- **KPI Работ (kpi_works):** Метрики по проектам — сроки, выполнение, проблемы
- **KPI Денег (kpi_money):** Финансовые KPI — выручка, расходы, маржа, долги

### 3.14. МОДУЛЬ «ФИНАНСЫ» (Finances)
**Путь:** `/finances`
**Роли:** ADMIN, BUH, Директора
**Описание:** Финансовая аналитика с двумя режимами — Расходы и Поступления.

**Категории рабочих расходов:** ФОТ (фонд оплаты труда), Логистика, Проживание, Трансфер, Химия, Оборудование, Субподряд, Прочее

**Категории офисных расходов:** Аренда, Коммунальные, Канцелярия, Связь, Транспорт, Хозтовары, ПО, Представительские

**Визуализация:** Столбчатые графики по месяцам (12 мес.), круговые диаграммы по категориям (SVG), карточки: итого, среднее, максимум. Навигация по годам.

### 3.15. МОДУЛЬ «СЧЕТА И ОПЛАТЫ» (Invoices)
**Путь:** `/invoices`
**Роли:** ADMIN, PM, BUH, Директора
**Описание:** Управление счетами. Создание, отслеживание оплат, частичные платежи. Связь с работами и актами. Статусы: Новый, Отправлен, Частично оплачен, Оплачен, Просрочен.

### 3.16. МОДУЛЬ «АКТЫ ВЫПОЛНЕННЫХ РАБОТ» (Acts)
**Путь:** `/acts`
**Роли:** ADMIN, PM, BUH, Директора
**Описание:** Генерация и учёт актов приёмки выполненных работ. Номер, дата, связь с работой, заказчик, описание, статус.

### 3.17. МОДУЛЬ «РЕЕСТР БУХ» (BUH Registry)
**Путь:** встроен в финансы
**Роли:** ADMIN, BUH, Директора
**Описание:** Полный финансовый реестр бухгалтера — все транзакции, счета, платежи в одном месте.

### 3.18. МОДУЛЬ «ОФИСНЫЕ РАСХОДЫ» (Office Expenses)

**Роли:** ADMIN, OFFICE_MANAGER, Директора

### 3.19. МОДУЛЬ «ИМПОРТ БАНКОВСКИХ ВЫПИСОК» (Bank Import)
**Путь:** встроен в финансы
**Роли:** ADMIN, BUH
**Описание:** Загрузка банковских выписок и автоматический парсинг для сверки платежей.

### 3.27. МОДУЛЬ «КАЛЬКУЛЯТОР РАБОТ» (Calculator)
**Путь:** встроен в PM Calcs
**Роли:** PM, ADMIN, Директора
**Описание:** Сложный калькулятор себестоимости работ (~35 КБ кода). Два варианта (v1 и v2).

**Рассчитывает:**
- ФОТ (фонд оплаты труда): дни × количество × ставки по ролям + подготовительная фаза
- Логистика: расстояние в км → автоматический подбор транспорта
- Проживание: суточные, СИЗ
- Химия: объём × цена за кг
- Оборудование: покупка / аренда / амортизация
- Мобилизация
- Накладные расходы (%)
- Налоги: ФОТ, прибыль, НДС

**Итоговые выводы:** Базовая себестоимость → Цена без НДС → Цена с НДС → Прибыль до/после налога → Прибыль на человека в день (валидация порога)

**Дополнительные справочники:**
- **calc_cities.js** (45 КБ): справочник городов с суточными, расстояниями
- **calc_equipment.js** (16 КБ): ставки аренды, амортизация оборудования
- **calc_norms.js** (14 КБ): нормы труда по видам работ, ролям, квалификациям

### 3.28. МОДУЛЬ «КАДРОВЫЕ ЗАЯВКИ» (HR Requests)
**Путь:** `/hr-requests`
**Роли:** ADMIN, PM, HR, Директора
**Описание:** Workflow заявок на персонал для работ.

**Процесс:**

## 4. РОЛИ И ПРАВА ДОСТУПА

Система имеет **11 ролей** с гранулированным контролем доступа:

### 4.1. ADMIN (Администратор)
**Полный доступ ко всем модулям.** Управление пользователями, настройками, диагностикой, Telegram-ботом, синхронизацией, бэкапами.

### 4.2. DIRECTOR_GEN (Генеральный директор)
**Доступ:** Все данные, все модули кроме технических (диагностика, sync, Telegram). Видит финансы, зарплаты, все тендеры/работы. Согласовывает премии, расходы, кадры.

### 4.3. DIRECTOR_COMM (Коммерческий директор)
**Доступ:** Тендеры, работы, финансы, прибыль. **Не видит:** зарплаты, кадровые данные.

### 4.4. DIRECTOR_DEV (Директор по развитию)
**Доступ:** Тендеры, работы, функции PM. Может выполнять роль PM параллельно.

### 4.5. PM (Руководитель проекта)
**Доступ:** Только свои тендеры/работы. Калькулятор, просчёты, управление бригадой, расходы по работам, запрос кадров, запрос закупок, бонусы. **Не видит** чужие проекты и финансы компании.

### 4.6. TO (Тендерный отдел)
**Доступ:** Все тендеры (но не работы), распределение тендеров на PM, воронка продаж, контрагенты.

### 4.7. HR (Кадры)
**Доступ:** Сотрудники, расписания, рейтинги, кадровые заявки, допуски. **Не видит** финансы и тендеры.

### 4.8. BUH (Бухгалтерия)
**Доступ:** Финансы, зарплаты, счета, акты, банковский импорт, работы (финансовая часть).

### 4.9. OFFICE_MANAGER (Офис-менеджер)
**Доступ:** Офисные расходы, корреспонденция, договоры, печати, доверенности, жильё и билеты.

### 4.10. WAREHOUSE (Складовщик)
**Доступ:** Склад ТМЦ, оборудование, QR-коды, перемещения.

### 4.11. PROC (Закупки)
**Доступ:** Закупочные заявки, согласование закупок.

### 4.12. PENDING (Ожидающий)
Новый пользователь, ещё не одобренный администратором. Нет доступа ни к одному модулю.

### Мульти-ролевой доступ
Система поддерживает **совмещение ролей** — один пользователь может иметь, например, PM + DIRECTOR_DEV. Предусмотрен переключатель ролей в интерфейсе.

## 5. БИЗНЕС-ПРОЦЕССЫ И ДВИЖЕНИЕ ЗАЯВКИ

### 5.1. Основной цикл: Тендер → Работа → Оплата

```
1. СОЗДАНИЕ ТЕНДЕРА (TO)
   │  Ввод: заказчик, ИНН, номер тендера, тип, описание, дедлайн
   │  Автопроверка на дубликаты (fuzzy matching)
   ▼
2. РАСПРЕДЕЛЕНИЕ НА PM (TO → PM)
   │  TO выбирает PM для просчёта
   │  PM получает уведомление
   ▼
3. ПРОСЧЁТ / СОЗДАНИЕ ТКП (PM)
   │  PM открывает калькулятор работ
   │  Рассчитывает: ФОТ, логистику, проживание, химию, оборудование
   │  Формирует ТКП с итоговой ценой
   ▼
4. СОГЛАСОВАНИЕ ТКП (PM → Директор)
   │  Директор проверяет и утверждает цену
   │  или отправляет на доработку
   ▼
5. ОТПРАВКА КЛИЕНТУ (TO/PM)
   │  ТКП отправлено → TKP Follow-up напоминает о контакте
   ▼
6. ОТВЕТ КЛИЕНТА
   │  Клиент согласился → Создаётся РАБОТА
   │  Клиент отказался → Тендер закрывается
   ▼
7. ЗАЯВКА НА КАДРЫ (PM → HR)
   │  PM указывает роли и количества
   │  HR подбирает людей из реестра
   │  PM утверждает состав бригады
   ▼
8. ЗАЯВКА НА ЗАКУПКИ (PM → PROC/WAREHOUSE)
   │  PM запрашивает оборудование/материалы
   │  Склад/Закупки обеспечивают
   ▼
9. ВЫПОЛНЕНИЕ РАБОТ (PM)
   │  Трекинг дат (план/факт)
   │  Учёт расходов
   │  Контроль бригады
   ▼
10. ЗАКРЫТИЕ РАБОТЫ (PM)
   │  Обязательная оценка каждого рабочего (1–10 баллов)
   │  Оценка удовлетворённости клиента (1–10)
   │  Формирование акта выполненных работ
   ▼
11. ФИНАНСОВОЕ ЗАКРЫТИЕ (PM → BUH)
   │  Выставление счёта
   │  Контроль оплаты (аванс + баланс)
   │  Бонусный запрос на бригаду
   ▼
12. СОГЛАСОВАНИЕ ПРЕМИЙ (PM → Директор)
   │  Директор одобряет → бонусы начисляются
   │  или отклоняет с комментарием
```

### 5.2. Движение документов
```
Входящие/Исходящие → Регистрация в корреспонденции (авто-нумерация)
Договоры → Реестр договоров → Связь с контрагентом
Доверенности → 7 шаблонов → Автозаполнение → Печать
Акты → Привязка к работе → Подписание → Счёт
```

### 5.3. Управление оборудованием
```
Поступление → Регистрация на складе → QR-код
Выдача → Назначение на работника/объект → Перемещение
Возврат → Проверка состояния → Склад или ремонт
Списание → Акт списания
```

## 6. ИНТЕГРАЦИИ

### 6.1. Telegram Bot
- **Уведомления** обо всех событиях CRM прямо в Telegram
- Привязка Telegram-аккаунта к профилю CRM (telegram_chat_id)
- Типы уведомлений: новые тендеры, согласования, дедлайны, сообщения чата
- Настройка через модуль `/telegram` (только ADMIN)

### 6.2. AI-ассистент «Мимир» (Mimir)
- **Виджет** в правом нижнем углу (64×64px, золотисто-красный градиент)
- **Стилистика:** Скандинавская/Викинг тема, руна ᛗ, «мудрость Одина»
- **Чат-интерфейс:** Тёмная панель, пузырьки сообщений (пользователь — красные справа, Мимир — синие слева)
- **Загрузка файлов:** PDF, Excel, изображения для анализа
- **RBAC (ролевая фильтрация данных):**
  - ADMIN/Ген.директор — видят всё
  - Ком.директор — тендеры, работы, прибыль (без зарплат)
  - PM — только свои тендеры/работы
  - TO — только тендеры
  - HR — только сотрудники
  - BUH — финансы и зарплаты
- **Примеры запросов:** «Сколько тендеров выиграли?», «Какие работы активные?», «Финансовый отчёт»

### 6.3. Манго Телеком (VoIP)
**Путь:** `/mango`
**Роли:** ADMIN
**Описание:** Интеграция с телефонией Манго Телеком. Карточка звонка при входящем, история звонков, привязка к контрагенту.

### 6.4. Email (SMTP/IMAP)
- Отправка уведомлений по email
- IMAP-интеграция для входящей почты
- Автоотчёты на email (ежемесячные, квартальные, годовые)
- Логирование отправок (email_log)

### 6.5. Синхронизация PostgreSQL
- Двусторонняя синхронизация (pull/push)
- Инкрементальная синхронизация по updated_at
- Конфликт-резолюция
- Индикатор статуса: idle, syncing, success, error, offline
- 20+ таблиц

## 7. ДОПОЛНИТЕЛЬНЫЕ ВОЗМОЖНОСТИ

### 7.1. PWA и офлайн-режим
- Установка как нативное приложение на любое устройство
- Service Worker кеширует все ресурсы
- IndexedDB для локального хранения данных
- Полная работоспособность без интернета
- Автоматическая синхронизация при появлении сети
- Уведомление о доступности обновления

### 7.2. Безопасность
- JWT-аутентификация с настраиваемым сроком (по умолчанию 7 дней)
- PIN-код как второй фактор (2FA)
- Bcrypt-хеширование паролей
- Rate-limiting: 10 запросов/сек, burst 20
- Nginx Security Headers: X-Frame-Options, CSP
- HTTPS с Let's Encrypt

## 10. РЕКОМЕНДАЦИИ ДЛЯ ПРЕЗЕНТАЦИИ

### Структура презентации (40–60 мин):
1. **Введение и обзор** (5 мин) — Что такое АСГАРД CRM, для кого, зачем
2. **Дизайн и UX** (5 мин) — Тёмная тема, скандинавская стилистика, PWA, мобильность
3. **Тендерный модуль** (7 мин) — Создание, воронка, распределение, дубликаты
4. **Калькулятор и ТКП** (7 мин) — Просчёт себестоимости, формирование цены
5. **Управление работами** (7 мин) — Жизненный цикл проекта, бригада, закрытие
6. **Финансы** (5 мин) — Расходы, доходы, счета, акты, банковский импорт
7. **HR и персонал** (5 мин) — Рейтинг, кадровые заявки, допуски, графики
8. **Оборудование и склад** (3 мин) — ТМЦ, QR-коды, перемещения
9. **Роли и безопасность** (5 мин) — 11 ролей, RBAC, аудит, 2FA
10. **Интеграции** (5 мин) — Telegram, AI Мимир, Манго Телеком, Email
11. **Офисные процессы** (3 мин) — Корреспонденция, договоры, печати, доверенности
12. **Аналитика и отчёты** (3 мин) — Дашборды, KPI, автоотчёты, экспорт

### Ключевые тезисы для презентации:
- **Всё в одном:** От тендера до оплаты в единой системе
- **Офлайн-first:** Работает без интернета (критично для полевых работ)
- **Ролевая модель:** 11 ролей, каждый видит только своё
- **AI-ассистент:** Мимир отвечает на вопросы с учётом роли пользователя
- **Автоматизация:** Автоотчёты, SLA, напоминания, уведомления
- **Скандинавский стиль:** Уникальная тематика, повышающая вовлечённость
- **Полный цикл:** Тендер → Просчёт → Работа → Акт → Оплата → Премия
- **Самодостаточная система:** Не требует дорогих лицензий (1С, Битрикс и т.д.)
